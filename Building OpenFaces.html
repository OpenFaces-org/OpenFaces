OpenFaces Project Structure

Here’s a directory structure of the OpenFaces project:

-          funcTests – automated Selenium-based tests for OpenFaces functionality.

-          lib – third-party libraries required for building and running OpenFaces modules.

-          libsrc – source code for some of the third-party libraries.

-          liveDemo

-          core – Source code for the OpenFaces LiveDemo. It is not a self-sufficient application, but it contains most of the demo code for creating two versions of the demo – Facelets and JSP versions.

-          facelets – Facelets-based version of OpenFaces LiveDemo.

-          jsp – JSP-based version of OpenFaces LiveDemo.

-          openFaces – Source code for the OpenFaces library itself.

-          seleniumInspector – A utility package for running Selenium-based tests. Used by the funcTests module.

-          testApp

-          core  – Source code for the OpenFaces Test Application. It is not a self-sufficient application, but it contains most of the application code for creating two versions of the application – Facelets and JSP-based versions.

-          facelets – Facelets-based version of OpenFaces Test Application.

-          jsp – JSP-based version of OpenFaces Test Application.

-          testLib – third-party libraries required for functional testing.

You may build OpenFaces project by several ways:

-          Using Eclipse IDE.

-          Using Intellij Idea IDE.

-          Using Ant build tool.

After building application you may find OpenFaces library archive and web application archives for running Demo and Test applications.

Configuring Eclipse

OpenFaces project requires Eclipse 3.4 or higher to run.

 The following configurations should be made in Workspace preferences to run OpenFaces:

1.      Install JDK version 1.5. Set Compiler Compliance Level to 1.5.

2.      Define Apache Tomcat v6.0 Server Runtime Environment that is referenced to Tomcat 6.0 installation folder.

3.      Set workspace Text file encoding preference to UTF-8 to ensure the correct interpretation of internationalized strings in source code.

4.      Define 'ProjectRoot' variable in Linked Resources preference page that links to the OpenFaces project's root folder in file system.

(For this purpose open Workspace Preferences dialog, type Linked Resources in quick search field, choose Linked Resources preference page, check the Enable Linked Resources check-box if unchecked, add new path variable named ‘ProjectRoot’ and choose project’s root folder location).

 

Configuring Idea

You should have IntelliJ IDEA 9.0.1 or higher to work with the OpenFaces project. Idea project file is named OpenFaces.ipr .

In order for project files taken from Git to work without any modifications the following system-wide Idea settings must be configured:

1.      There must be a JDK version 1.5.0_x in Idea named "1.5".

2.      Configure the Tomcat 6.0.x server named "Tomcat 6".

3.      File encoding should be set to UTF-8 to ensure correct interpretation of internationalized strings in source code (configurable on the Settings | Project Settings | File Encodings page of Idea settings).

 

Using Ant Scripts

There is an ability to build the OpenFaces library, Test Application and Live Demo using Ant scripts. To do this, find build.xml file in project’s root folder and run the default target named build. The result of executing the script is OpenFaces library file and several WAR files.

The OpenFaces library jar file may be found at <ProjectRoot>\openFaces\target\openfaces.jar.

WAR files for Demo and Test applications can be found in the following directories:

-          <ProjectRoot>\liveDemo\facelets\target\

-          <ProjectRoot>\liveDemo\jsp\target\

-          <ProjectRoot>\testApp\facelets\target\

-          <ProjectRoot>\testApp\jsp\target\

 

Running Tests

You may also run functional Selenium tests after building and deploying test and demo applications.

Here's a list of ant targets that can be run from the project's root folder for running tests:

run-func-tests-facelets

Runs Selenium tests for Facelets-based version of test and demo applications.
Requirements: Installed Mozilla Firefox, test and demo applications deployed under TestAppFacelets_3x and LiveDemoFacelets_3x contexts accordingly. These contexts may be defined at runtime with test.app.context.path and demo.context.path system properties.

run-func-tests-jsp

Runs Selenium tests for jsp version test and demo applications.

Requirements: Installed Mozilla Firefox, test and demo applications deployed under TestAppJsp_3x and LiveDemoJsp_3x contexts accordingly. These contexts may be defined at runtime with test.app.context.path and demo.context.path system properties.

By default applications must be deployed under http://localhost:8080. To change this setting find funcTests.properties file at <ProjectRoot>\funcTests\source\ folder and change org.openfaces.funcTests.startUrl property.The  funcTests.properties file also contains the org.openfaces.funcTests.firefox.path property that specifies a location of the Firefox browser.

By default applications must be deployed under the following contexts:

-          LiveDemoFacelets and TestAppFacelets for Facelets-based versions,

-          LiveDemoJsp and TestAppJsp for JSP-based version.

The following Ant target may be found useful:

clean

Removes the "target" directories generated during the build process. This target should be invoked if you have to clean target directories.
